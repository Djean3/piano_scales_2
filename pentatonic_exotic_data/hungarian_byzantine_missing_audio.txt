Missing voice-file audio for Hungarian Minor + Byzantine scale data
=====================================================================
Generated by pentatonic_exotic_data/_generate.js (checked every voiceAudio
path this build emits against a Glob of voices/*.wav). All 9 gaps below stem
from the same root cause: finger-5 (pinky) voice clips only exist for the 7
natural/white notes (f5_A/B/C/D/E/F/G.wav) — there are NO f5_<accidental>.wav
files anywhere in this project yet. Two more one-off gaps come from finger-3
missing "As"/"Fb" and finger-4 missing "Cb" (pre-existing gaps in the voice
library, not something introduced by this build).

The generated hungarian_byzantine.js still WRITES these paths (per the task
spec) — they just won't play audio until the corresponding .wav files are
recorded/added to voices/.

f5_Cs.wav
  Used by: Db Hungarian Minor (RH), Db Hungarian Minor (LH)
  Why: Db Hungarian Minor's root is spelled C# (this app's established
  practical spelling for Db-minor-family scales — see FINGERING_REVIEW_LOG.md
  precedent / Db Natural Minor in data.js). C# is a black key, so it never
  gets a finger-5 clip under the existing recording scheme.

f5_Db.wav
  Used by: Db Byzantine (RH), Db Byzantine (LH)
  Why: same root-note issue as above, but Db Byzantine keeps the flat
  spelling "Db" (no double-flat problem for a major-scale-derived scale),
  so it hits the missing f5_Db.wav instead of f5_Cs.wav.

f5_Eb.wav
  Used by: Eb Hungarian Minor (RH), Eb Hungarian Minor (LH),
            Eb Byzantine (RH), Eb Byzantine (LH), Bb Byzantine (RH)
  Why: Eb Hungarian Minor / Eb Byzantine's root (Eb) is black. Bb Byzantine
  also passes through an Eb at a non-root scale degree that happened to land
  on finger 5 under the reconstructed fingering.

f5_Fs.wav
  Used by: F# Hungarian Minor (RH), F# Hungarian Minor (LH),
            F# Byzantine (RH), F# Byzantine (LH)
  Why: F# Hungarian Minor / F# Byzantine's root (F#) is black.

f5_Ab.wav
  Used by: Ab Hungarian Minor (RH), Ab Hungarian Minor (LH),
            Ab Byzantine (RH), Ab Byzantine (LH)
  Why: Ab Hungarian Minor / Ab Byzantine's root (Ab) is black.

f5_Bb.wav
  Used by: Bb Hungarian Minor (RH), Bb Hungarian Minor (LH),
            Bb Byzantine (LH)
  Why: Bb Hungarian Minor's root (Bb) is black.

f3_Fb.wav
  Used by: Eb Byzantine (RH), Ab Hungarian Minor (RH), Ab Hungarian Minor (LH),
            Ab Byzantine (RH), Ab Byzantine (LH)
  Why: this app's voice library already has f1_Fb / f2_Fb / f4_Fb but never
  recorded f3_Fb — a pre-existing gap, not new to this scale type. Several
  of these scales' fingering happens to land finger 3 on an Fb-spelled note.

f3_As.wav
  Used by: F# Byzantine (LH)
  Why: same pre-existing gap pattern — f1_As/f2_As/f4_As exist, f3_As
  doesn't. Only one scale/hand in this build happens to need it.

f4_Cb.wav
  Used by: Ab Hungarian Minor (RH), Bb Byzantine (LH)
  Why: f1_Cb/f2_Cb/f3_Cb exist but f4_Cb doesn't (pre-existing gap).

Summary: 9 distinct missing combos, all pre-existing library gaps (finger-5
never recorded for any accidental note; finger-3 missing As/Fb; finger-4
missing Cb) surfaced by these two new scale types rather than caused by them.
