`_missing_audio.txt`, and `_needed_audio_names.txt` from scratch — the

Not wired into `data.js` / `index.html` / `app.js` — output only, per

## 1. Source material and what it actually gave us

- `pianoscales.org/hirajoshi.html` and `.../spanish-gypsy.html` were
  fetched for the pitch content (rate-limited but reachable both times,
  no 429s hit this session).

  (`whole_tone_diminished_review.md`) found that this site's *fingering*
  is sometimes present directly in the raw page HTML even when the

  "fingering is member-only" premise on faith, I pulled the **raw HTML**

  any summarization) and checked myself. Confirmed: for these two scale
  types specifically, the premise was correct — neither page has a
  `Fingering (LH)/(RH)` block anywhere in the markup, only a `Notes:`
  line per key (checked both pages in full, not just spot-checked).
  So fingering really is unavailable here, unlike the whole-tone case.
- The raw HTML **did** give an unexpected bonus: a full per-key
  "Scales overview" block at the bottom of each page listing all 12
  keys' note spellings, including alternate sharp/flat spellings and
  the site's own parenthetical enharmonic resolutions (e.g. "Bbb (A)").
  This let me cross-check all 24 of this build's computed note lists
  against the site's own published values directly, not just the 3
  ground-truth keys the task specified.
  - **Result: 23 of 24 note-lists match the site byte-for-byte**
    (mod notation, e.g. their "E# (F)" vs. this build's plain "F").
    This includes the Db-substitution decision (§2) — the site's own
    flat-alternate row for C#/Db independently hits the *same*
    double-flats (Ebb, Bbb) this build ran into, and resolves them
    parenthetically to the *same* natural notes (D, A) this build's
    fallback logic produced, without me having seen the site's row
    beforehand.
  - **The 1 mismatch:** the site's G#/Ab Spanish Gypsy flat-alternate
    row lists degree 2 as "Bb", but its own sharp-primary row for the
    *same key* lists degree 2 as "A" — and "A" is what the ♭2-of-Ab
    formula actually produces (Ab root + 1 semitone = A, verified
    arithmetically). This build kept "A" (matching the formula and the
    site's own sharp row) rather than the flat row's apparent
    transcription slip; flagged here rather than silently overridden.
- A WebSearch cross-check ("hirajoshi scale piano fingering",
  "phrygian dominant scale piano fingering spanish gypsy") turned up
  musmath.com, pianoencyclopedia.com, muted.io, and Wikipedia — all
  confirm the note content (e.g. D Phrygian Dominant = D, Eb, F#, G, A,
  Bb, C, matching `D_SPANISH_GYPSY` here) but **none published actual

  major/minor fingering patterns apply," thumb starts on the tonic).
  No book or site gives an authoritative finger-by-finger answer for
  either scale in any key.
- **Bottom line: pitches/note-names are now high-confidence** (formula
  + raw-HTML page content for all 24 entries, not just 3 + WebSearch
  agree). **Fingering remains a from-scratch reconstruction with no
  book/site precedent** — same category as this project's own
  "flagged for human re-check" entries in `FINGERING_REVIEW_LOG.md`.
  Treat any specific finger# as provisional until played through by
  hand; treat any specific note/pitch as solid.

## 2. Pitch/spelling method (high confidence)

Both scales were built as alterations of each key's own major scale
degrees, using the same 12-key anchor set already in `data.js` (RH
tonic = `60 + semitone-offset-from-C`, LH tonic = RH − 12, read
directly from each key's own `"{Key} Major"` entry — confirmed
identical for all 12 keys, i.e. LH is always exactly one octave below
RH in this app).

- **Hirajoshi** = major-scale degrees 1, 2, ♭3, 5, ♭6 (drop 4 and 7).
  Verified against the task's three given ground-truth keys (C, G, A)
  and pianoscales.org — exact match in all three.
- **Spanish Gypsy / Phrygian Dominant** = major-scale degrees 1, ♭2, 3,
  4, 5, ♭6, ♭7 (5th mode of harmonic minor). Verified against the
  task's C example and against `pianoscales.org` / WebSearch's D
  example — exact match.
- Flattening was computed from each key's **true, unsimplified** major
  scale (e.g. F♯ major's real 7th degree, E♯, not the app's
  already-simplified "F" display) and only simplified back down at the
  very end, so degrees that happen to land on E♯/B♯ collapse to F/C
  (matching this project's existing F♯-major precedent), while Cb/Fb
  results are **kept as-is** (matching the existing Eb-minor/Ab-minor
  precedent of legitimately using Cb/Fb rather than always chasing a
  natural-letter spelling).

### Db substitution (mirrors the existing "Db Minor plays C♯ minor" precedent)

Db's own flat spelling produces a genuine double-flat for two degrees
(Hirajoshi's ♭6 = Bbb; Spanish Gypsy's ♭2 = Ebb) — the same kind of
problem this app already solved for "Db Minor" by playing C♯ minor's
notes under the Db label (documented in `FINGERING_REVIEW_LOG.md`,
"Db Major + 'Db Minor' family"). Both new Db-labeled scales in this
build follow the identical pattern: **computed from C♯'s sharp
spelling internally, labeled "Db" in `name`/`nameAudio`, root note
pitched/spelled as C♯ (`f1_Cs.wav`, etc.)** — consistent with how
`Db Natural Minor` already spells its own root as `f2_Cs.wav`, not
`f2_Db.wav`. Result:

- `Db Hirajoshi` = C♯, D♯, E, G♯, A
- `Db Spanish Gypsy` = C♯, D, F, F♯, G♯, A, B

### Ab's one single-degree patch

`Ab Spanish Gypsy`'s ♭2 also lands on a double-flat in Ab's own native
table (Bb → Bbb). Unlike Db, switching Ab's whole scale to its
sharp-side enharmonic (G♯) does **not** help — G♯ major itself needs 8
sharps (including a theoretical double-sharp on its own 7th degree),
which is worse, not better. So this one degree only was patched locally
(Bbb → A natural, verified pitch-class-correct), while every other
degree of `Ab Spanish Gypsy` keeps Ab's native flat spelling — this
matches how `Ab Minor` already keeps its own Cb/Fb natively rather
than being wholesale-substituted. Result: `Ab Spanish Gypsy` = Ab, A,
C, Db, Eb, Fb, Gb. (`Ab Hirajoshi` needed no patching at all — its
worst case is a plain single-flat Cb/Fb, both already-established
spellings in the Ab family.)

Every other key (C, D, Eb, E, F, F♯, G, A, Bb, B) resolved cleanly with
plain single-accidental-or-natural spellings straight from that key's
own native major scale — no substitution or patching needed.

## 3. Fingering method (reconstructed — no precedent)

### Spanish Gypsy (7 notes + octave = 8 ascending slots)

Directly reused each key's **own already-built Major scale's RH/LH
finger-per-scale-degree-index shape**, applied to Spanish Gypsy's own
(different) pitches at the same index positions. This is the identical
technique this project already used for Dorian/Phrygian/Lydian/
Mixolydian/Locrian in every key (see `FINGERING_REVIEW_LOG.md`,
"Modes... for all 12 roots" — "reuses its OWN root's already-confirmed
major-scale hand shape verbatim"). Descending fingers mirror ascending
per-pitch (same rule the whole app already uses everywhere: a given
pitch always gets the same finger regardless of direction).

Because this reuses shapes already vetted for 58 other scales, and
because finger 5 in every one of those shapes only ever lands on a
natural-letter note (see §4), Spanish Gypsy's fingering carries the
**same confidence level as the modes did** — i.e. "reasonable, reused,
not independently re-verified for this specific new scale," a couple
notches more solid than Hirajoshi's fully-from-scratch fingering below.

### Hirajoshi (5 notes + octave = 6 ascending slots) — fully invented, no reuse possible

A 5-note scale doesn't fit any existing 7-note (or 8-slot) scaffold in
this app, so this fingering has zero precedent to lean on, reused or
otherwise. Constructed as follows and should be treated as the
**lowest-confidence part of this whole build**:

- **RH ascending:** 1, 2, 3 on the root/2nd/♭3, then a "thumb restart"
  back to 1, 2, 3 on the 5th/♭6/octave. (Six slots, two mirrored
  3-note hand positions — reasoned from the observation that Hirajoshi's
  five notes span less than an octave and a half comfortably, so no
  finger-5 stretch is needed; the restart point is a genuine, un-cued
  physical hand shift since `cross: null` throughout this pass.)
- **RH descending:** exact mirror, same finger per same pitch (b6=3,
  5th=2, ♭3=1... — matches the app's own "same pitch, same finger"
  rule used everywhere else).
- **LH:** built as the mirror-image complement of RH (finger `f` on RH
  ↔ finger `5−f` on LH, i.e. 1↔4, 2↔3), deliberately keeping LH in the
  1–4 range only (see §4 for why).
- `now_backwards` slot always repeats the last ascending note's pitch
  and finger, matching the `C_MAJOR` template exactly.

This is a defensible, standard-technique construction (thumb-based
hand positions, mirrored hands, no wide stretches) but it is **not**
verified against any book, site, or forum — if you find an authoritative
Hirajoshi fingering later, this is the part most likely to need
revision, not the pitches.

## 4. Why finger 5 was avoided almost everywhere

Checked the full voice-file inventory in `voices/` before designing
fingering:

| finger | accidental coverage |
|---|---|
| f1 | complete (all 12 accidental spellings + 7 naturals) |
| f2 | complete |
| f3 | missing `As`, `Fb` |
| f4 | missing `Cb` |
| f5 | **naturals only — zero accidental files exist** |
| note_ (together section) | complete |

Cross-checking this against all 58 already-built scales in `data.js`
turned up a consistent pattern: **finger 5 is never assigned to an
accidental note anywhere in this app** — every existing use of finger 5
lands on a natural letter (C/D/G/A/E/B tonics or octaves), and six of
the twelve major keys (F♯, F, Bb, Eb, Ab, Db) avoid finger 5 entirely
in one or both hands specifically because their shapes would otherwise
put it on a black key. That's clearly a deliberate, standing rule in
this app's audio design, not an accident — so Hirajoshi and Spanish
Gypsy's fingering follows the same rule: **finger 5 is not used at all
in either new scale type, in any key.** Spanish Gypsy achieves this
"for free" (it's inherited from Major shapes that already respect the
rule); Hirajoshi's from-scratch design (§3) was built around it
directly.
